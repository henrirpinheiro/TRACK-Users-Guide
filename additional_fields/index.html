    Calculation of Additional Fields

    Additional Track File Attributes

    You can append one or more fields to the tracks, provided that the data are on the same grid as the original dataset and maintain the same format (i.e., matching time steps and periods). Several sampling methods are available for adding new data to the tracks:

    • Input 0 → Basic interpolation (max/min at feature points)
    • Input 1 → Area averages or variance
    • Input 2 → Regional radial or lat-lng values
    • Input 3 → Sampling at fixed locations

    Area Average¶

    Area averaging is commonly used for precipitation, as it ensures that a sufficient area is covered to represent the storm-associated rainfall. When using this method, you need to specify outer and inner search distances, which define the maximum range over which the algorithm will compute the new average value.

    For example, in the input file below, the outer and inner values are set to 6.0° and 5.0°, respectively. You can also sample values at a point offset from the actual track feature (e.g., vorticity or MSLP).

    ➕ Download add_prec_era5.in

    The added field can be scaled as needed. In the example, precipitation values are scaled by 3600. This conversion accounts for the fact that ERA5 total precipitation data are provided in kg m⁻² s⁻¹, which must be multiplied by 3600 to convert to mm per hour.

    To calculate additional fields, use the following command:

    bin/track.linux < add_prec_era5.in

    This will append precipitation data to the tracks based on the configurations defined in the input file, and an output file will be created in the output directory named ff_trs.linux_addfld, which should be renamed accordingly.

    Basic Interpolation¶

    When using basic interpolation (input 0), you need to choose how values are sampled relative to the feature point, which can be:

    • At the feature point (input 0)
    • Minimum (input -1)
    • Maximum (input 1)

    Additionally, you have to select the search method for extrema:

    • Spline interpolation with steepest ascent/descent (input 0)
    • General search (input 1)

    Finally, you must specify outer and inner search distances, similar to area averaging. The syntax is the same as that used for area average.

    Note: The add_prec_era5.in file is specifically designed for ERA5 reanalysis data and should be adjusted accordingly when working with other datasets. Before running the full process, it's strongly recommended to test the configuration manually to ensure it behaves as expected. To do this, execute the following command:

    bin/track.linux

    You will then be prompted to enter each line of the configuration interactively, which helps ensure all values are correctly set for your dataset.
