    Cyclone Spatial Structure Composites

    üåç Synoptic-Scale Composites of Cyclonic Systems

    To investigate the synoptic-scale structure of weather systems such as cyclones, horizontal and vertical composites can be constructed using the methodology outlined by Catto et al. (2010, Journal of Climate). These spatial composites can be computed on pressure or potential temperature surfaces, or for single level fields (e.g., mean sea level pressure or precipitation). Vertical cross-sections through different cyclone features are also supported.

    Composites can be generated for the time of maximum intensity based on a selected metric, or at time steps offset from this peak‚Äîallowing you to examine both the development and decay phases of the system.

    üõ†Ô∏è Step 1: Generate Additional Fields¶

    Start by calculating additional field values around each cyclone. Fields can be sampled using either:

    • A radial grid, centered on the North Pole, with a prescribed radius (e.g., 15¬∞ or 20¬∞), or
    • A rotated rectangular lat-lng grid, centered on the system and aligned with the equator

    If using the rectangular grid method, specify:

    • Grid spacing (e.g., 0.5¬∞)
    • Number of points (e.g., 100 in both directions)

    Optionally, the grid can be rotated to align with the direction of cyclone motion, allowing to compute composites based on system-relative winds.

    To extract a field such as mean sea level pressure (MSLP) around each system, use:

    bin/track.linux < add_mslp_era5.in

    This will produce a file named ff_trs.linux_addfld_reg in the outdat directory.
    Rename the output (e.g., reg_MSLP_MAM1979) to avoid overwriting files when repeating the process across seasons or years.

    üì• Download add_mslp_era5.in

    üß© Step 2: Combine Seasonal/Yearly Files¶

    After generating all individual reg files (e.g., 168 for 42 years √ó 4 seasons), merge them into a single file:

    utils/bin/combine_reg < combine_reg.in

    This merges multiple seasonal files into one (e.g., combined_reg), which should be renamed (e.g., reg_MSLP_ERA5) for clarity.

    üì• Download combine_reg.in

    üåÄ Step 3: Generate Composite Fields¶

    Use the tcident tool to compute composite fields:

    utils/bin/tcident < tcident.in

    You can generate composites at:

    • Time of maximum or minimum intensity
    • Time of intensity tendency extrema
    • Maximum vertical gradient
    • A specific time step in the cyclone lifecycle

    You may define your own reference variable (e.g., vorticity or MSLP), though a default is typically used.

    üì• Download tcident.in

    Output: A NetCDF file containing composite fields will be created in the same directory as the reg files.
