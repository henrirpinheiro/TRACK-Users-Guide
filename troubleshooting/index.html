    Common Errors and Solutions

    🛠️ Troubleshooting

    1. Fixing Segmentation Fault in track.linux¶

    If running any task using:

    bin/track.linux

    results in a crash with the message "Segmentation fault", it's likely due to a shell-related issue.

    Solution:
    To resolve it, increase the stack size limit:

    ulimit -s unlimited

    Then rerun the command.

    2. Resolving the Master Lock File Issue¶

    If you encounter the following message:

    A master lock file has been detected, possible causes:
     (a) The master shell script is currently running.
     (b) A redundant lock file exists due to an aborted master script.
Do you want to reset the lock or wait for it to be released?
         0 ----  wait.
         1 ----  reset.
Waiting for the master lock to be released.

    Solution:
    This is likely due to leftover .master.lock files. List and remove them:

    ls -al .master.lock*
rm .master.lock*

    After this, the program should run normally.

    3. Resolving Incorrect Tracking and Vorticity Scaling¶

    If output shows unrealistic or constant vorticity values like:

    1 202.724106 -10.558068 7.706274e+16 
2 202.724106 -10.558068 7.706274e+16 
...

    Solution:
    This typically means there are missing values in the U and V wind fields. Use a visualization tool like xconv (available in the converters/ directory) to inspect your data.

    If missing values are found, fill them with:

    cdo fillmiss infile outfile

    This should correct the issue and ensure proper tracking and intensity scaling.

    4. Disabling rsh in the Master Script¶

    If you are prompted for a password when running the TRACK master script, it's because rsh is enabled. To disable it:

    Step 1: Edit the master script.
    Find this line:

    RSH="rsh -n"

    Replace it with:

    RSH="norsh"

    Step 2: Modify NORSH assignment logic.
    Replace this block:

    rsh $HOST ls >> /dev/null
if [ $? != 0 ]
then
   echo "no working rsh to $HOST using job submission to current machine without rsh"
   NORSH="norsh"
else
   echo "Have working rsh to $HOST"
   NORSH="rsh"
fi

    with:

    # Disable rsh checks and force "norsh" mode
NORSH="norsh"

    This prevents rsh prompts and forces job submission in "norsh" mode.
