    Combining Tracks and Producing Spatial Field Estimation

    Combining Track Files and Producing Spatial Fields with Kernel Estimators

    Spatial statistics are produced from the tracks using spherical kernel estimators (Hodges 1996). Tracks are first grouped by months or seasons using the combine utility, located in the utils directory. Seasons follow the standard meteorological definitions: DJF (Decemberâ€“February), MAM (Marchâ€“May), JJA (Juneâ€“August), and SON (Septemberâ€“November). The combine utility merges track files for a given season across multiple years, creating a single file representing the entire period. For example, all track files for MAM across available years are combined to compute a seasonal mean.

    To combine track files, run the following command in the main directory:

    utils/bin/combine < combine.in

    This generates an output file called (combined_tr_trs) which should be renamed accordingly (e.g., tr_trs_ERA5_MAM1979-2020). Repeat this process for each season.

    Example combine.in file:

    42
1
/path_to_the_track_files/ff_trs_MAM1979
/path_to_the_track_files/ff_trs_MAM1980
. . .
/path_to_the_track_files/ff_trs_MAM2020

    The first line specifies the number of track files to combine (42 in this case). The second line defines the time scaling:

    • 1 = Seasonal
    • 2 = Monthly
    • 3 = Daily

    Since we are working with seasonal data, we use 1.

    The next step is to generate the spatial statistics using the bb_stats.job executable. Before running this, ensure that the all_pr_in file is present in the same directory as the track files.

    Example all_pr_in file:

    #!/bin/csh
set FIELD = $1
set DIR=$2
set BASE=$3
set VAR=tr_trs_$FIELD
set STUB=$FIELD
if( ! -e ${DIR}/STATOUT_$STUB) then
   run_stats -c=$DIR -d=STATS.latlng.in -e=track.linux -f=era -i=tr_trs_$STUB -o=${BASE}
endif

    This script calls a file named STATS.latlng.in, which must be located in the indat directory.

    Once the required files are in place, run the following command in the main directory:

    bb_stats.job track_file[without tr_trs_] subdirectory path

    For example, if the track file produced in the previous step is stored in /media/user/TRACKS, use:

    bb_stats.job ERA5_MAM1979-2020 TRACKS /media/user

    This process may take some time, depending on the file size and available computational resources.

    The output will be a NetCDF file named:

    stat_trs_scl_ERA5_MAM1979-2020_1.nc

    This file will be located in the same directory as the track file. However, certain software (e.g., GrADS) may not be able to read the file directly. If needed, you can reformat it using the Climate Data Operators (CDO) tool with the following command:

    cdo -f nc copy stat_trs_scl_ERA5_MAM1979-2020_1.nc stat_trs_scl_ERA5_MAM1979-2020_new_file.nc

    This process generates a NetCDF file containing various spatial statistical fields, as shown below. The file has an X-Y dimension with a fixed resolution of 128 points in X and 64 points in Y, regardless of the original data resolution used to produce the trajectories. This ensures easier comparison between datasets of different resolutions.

    Spatial Statistical Fields:

    • mstr = Mean Intensity
    • stdstr = Std of Intensity
    • msp = Mean Speed
    • stdsp = Std of Speed
    • fden = Feature Density
    • gden = Genesis Density
    • lden = Lysis Density
    • tden = Track Density
    • xvel = X-component of Mean Velocity
    • yvel = Y-component of Mean Velocity
    • mlif = Mean Lifetime
    • mgdr = Mean Growth/Decay Rate
    • miso = Mean Anisotropy
    • xor = X-component of Mean Orientation Vector
    • yor = Y-component of Mean Orientation Vector
    • mten = Mean Tendency
    • marea = Mean Area
    • spare1 = Spare1
    • spare2 = Spare2
