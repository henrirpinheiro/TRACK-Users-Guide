    Producing Track Files with the Master Executable

    Producing Track Files with the Master Executable

    Once the pre-processed vorticity data is ready, you can generate the track files using the master script. This step requires careful attention to the syntax:

    master -c=VOR850_MAM1979_T42 -d=now -e=track.linux -i=ERA5_VOR850_MAM1979_filt.dat -f=MAM1979 -n=1,32,12 -o={path_where_the_track_files_will_be_produced} -j=RUN_AT.in -r=RUN_AT_ -s=RUNDATIN.ERA5_VOR

    Here's a breakdown of the options:

    • -c=: Name of the directory that will contain the output files
    • -d=: Stores current date and time (use now)
    • -e=: Name of the main executable (e.g., track.linux)
    • -i=: Input data file (produced in the previous filtering step)
    • -f=: Label for the run (e.g., MAM1979)
    • -n=: Time settings (e.g., 1,32,12 for seasonal analysis:
    • 1: Start time
    • 32: Approximate days per month
    • 12: Total time steps (3 months × 4 time steps/day))
    • -o=: Output path where track files will be saved
    • -j=: Syntax file for run control (e.g., RUN_AT.in)
    • -r=: Run name prefix (e.g., RUN_AT_)
    • -s=: Input file for controlling intensity threshold (e.g., RUNDATIN.ERA5_VOR)

    📄 Output Files¶

    The code creates text files with information on the location (longitude and latitude) and intensity for each track at each time step in radial coordinates.

    By default, it produces the following compressed files:

    • tr_trs_neg and tr_trs_pos:
      Contain all identified tracks for the Southern and Northern Hemispheres, respectively.
      Each track includes the sequence of feature points that meet the vorticity (or MSLP) threshold.

    • ff_trs_neg and ff_trs_pos:
      Filtered versions of the above, excluding tracks that don't meet the criteria set in RSPLICE.in for minimum lifetime and travel distance.

    Note:
    - The suffix "neg" indicates Southern Hemisphere tracks.
    - The suffix "pos" indicates Northern Hemisphere tracks.

    On average, generating the output track files for one season and one year may require approximately 3.5 MB, depending on resolution.

    📘 Example Output Format¶

    0
0 0
TRACK_NUM 1148 ADD_FLD 0 0 &
TRACK_ID 3
POINT_NUM 6
1 231.915375 -16.565212 2.238685e+00
2 231.581451 -18.106936 1.992483e+00
3 230.683823 -18.975389 1.859596e+00
4 230.880386 -18.826279 2.329082e+00
5 231.136749 -19.203224 2.855023e+00
6 231.395935 -20.242941 1.929559e+00
...

    This means: - TRACK_NUM: There are 1148 tracks - TRACK_ID: The reference number of the track (e.g., 3) - POINT_NUM: Number of points in the track (e.g., 6 time steps = 36 hours lifetime)

    Columns in each point line:

    1. Timestep
    2. Longitude
    3. Latitude
    4. Intensity (e.g., T42 850-hPa relative vorticity, scaled by -1.0 × 10⁻⁵ s⁻¹ for SH)
