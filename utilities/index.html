    Utility Programs

    Track Processing Utilities: Box / Count / Censemble2

    The TRACK algorithm provides a range of utilities for post-processing trajectories, including:

    • Spatial and temporal domain selection
    • Track comparison between different datasets
    • Frequency distributions of intensity and associated variables (e.g., area-averaged precipitation)

    These tools are located in the utils directory.

    üì¶ Box: Select trajectories within a latitude-longitude domain¶

    utils/bin/box [filename] [Lat1] [Lat2] [Lng1] [Lng2] [Genesis (0)/Lysis (1)/Passing (2)/Passing Time (3)/All Times (4)] [Negate (1)] [Track Thresh] [Start Time, YYYYMMDDHH]
    • Lat1, Lat2: southernmost and northernmost latitude bounds
    • Lng1, Lng2: westernmost and easternmost longitude bounds

    Example:

    utils/bin/box track_file -60 -40 280 330 0 0

    This command generates a new file in the same directory as the original, with the extension .new. The output will contain only the tracks that originated within the specified domain: latitudes between -60¬∞ and -40¬∞, and longitudes between 280¬∞ and 330¬∞.

    üß≠ Count: Select trajectories within a spherical region¶

    utils/bin/count [filename] [Lat.] [Lng.] [Radius] [Genesis (0)/Lysis (1)/Passing (2)/Passing Time (3)/All Times (4)] [Negate (1)] [Start Time, YYYYMMDDHH] [Timestep]

    This command filters tracks to retain only those whose genesis points lie within a spherical cap (defined in geodesic degrees).

    Example:

    utils/bin/count track_file -35.0 280.0 8.0 0 0

    This selects tracks that originated within a circular region of 8 degrees radius, centered at 35¬∞S, 80¬∞W. The output file, named with a .new extension, will be saved in the same directory.

    ‚è± Time Conversion¶

    The count command supports converting time steps into real-time format (e.g., YYYYMMDD). Use the following syntax:

    utils/bin/count track_file 0 0 5 4 0 1979030100 6
    • 1979030100: reference start time
    • 6: time step (in hours)

    üî¨ Censemble2: Advanced filtering and comparison of tracks¶

    censemble2 [track_file1] [track_file2] [temp1] [temp2] [nbin] [mean (0), max (1) or min (2) intensity : offset] \
[Mean (0) or Min (1) separation : use whole track (0) or start of track (1)] [format of main distributions #/month (0), frequency (1), pdf (2)] \
[Add. field Id. 1: location : time sample] [Add. field Id. 1 : use location (0 or 1) : use time sampling] \ [Add. field Id. 2 : use location : use time sampling]
[Ftype: 's' or 'vn', n=0 (speed), n=1 (tendency), n=2 (growth)] [Region matching type, '-1' (no region matching), '0' (track in region), '1' (extrema in region), '2' (60% of points in region),'3' (extrema of region)] \
[Re-order tracks, '1' for yes or '0' for no] [Dist. Thresh. (optional)] [Time. Thresh. (optional)][Intensity Range, I1, I2 (optional)]

    üìä Frequency Distribution¶

    The censemble2 utility is used to generate frequency distributions of any variable present in the track file which will be displayed on the prompt. Below is an example command and a breakdown of its syntax:

    utils/bin/censemble2 track_file track_file 0 28 10 1 0 1 0 0 s -1

    Explanation of each argument (in order):

    First term (track_file): Input track file.

    Second term (track_file): The same track file must be provided again.

    Third term (0): Minimum value of the intensity range (e.g., minimum vorticity).

    Fourth term (28): Maximum value of the intensity range.

    • ‚ö†Ô∏è If a track exceeds this range, you‚Äôll see:
    ***ERROR***, strength outside range for a track number

    Fifth term (10): Number of bins for the frequency distribution.

    Sixth term (1): Statistic used for intensity:

    • 0 = Mean
    • 1 = Maximum
    • 2 = Minimum

    Seventh term (0): Mean or minimum separation distance (used for matching tracks).

    Eighth term (1): Distribution format (month, frequency, or PDF).

    Ninth and Tenth term (0): Field selection flag (used if there are additional fields). Set both ninth and tenth terms to 0 if not using extra fields.

    Eleventh term (s): Smoothing option. Use s for standard smoothing.

    Twelfth term (-1): Region matching type:

    • -1 = No region.dat used
    • Positive integers refer to region groupings in region.dat if used

    üìç Region and Time Selection¶

    utils/bin/censemble2 track_file track_file 0 1000 10 0 0 0 0 0 s 2 0

    Requires a file named region.dat in the same directory.
    üì• Download region.dat

    Using the example input file, select tracks where longitude spans 280¬∞‚Äì350¬∞ and latitude spans -45¬∞ to -20¬∞, with ‚â•60% of their points falling within this region, and where the time range (for time step) is between 1 and 1,000,000.

    ‚ö†Ô∏è Warning: Remove region.dat after use to avoid unintended filtering.

    üå™ Extracting Maximum Intensity Locations¶

    Use the str.dat file generated by the Censemble2 to extract maximum intensity location. Example for 95th percentile:

    awk '{print $9}' str.dat | sort -n | awk '{all[NR] = $0} END{print all[int(NR*0.95 - 0.5)]}'

    üîÅ Matching Tracks Between Datasets¶

    To match the same systems between two datasets (e.g., vorticity and MSLP):

    utils/bin/censemble2 track_file1 track_file2 0 1000 10 1 0 0 0 0 s -1 4.0 0.5
    • 4.0: max displacement (deg)
    • 0.5: minimum percent of overlapping points (50%)

    Latitude/longitude will follow the first track file.

    To compute the mean separation distance, reverse the files and run the command again:

    utils/bin/censemble2 [track_file2] [track_file1] 0 1000 10 1 0 0 0 0 s -1 4.0 0.5
